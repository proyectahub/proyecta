## Compound library assembly for virtual screening

A docking-oriented repurposing library was assembled from the DrugCentral `FDA_Approved.csv` registry and the current DrugCentral 2023 structural archive (`structures.molV2.sdf.gz`). Only compounds with an explicit recoverable chemical structure were retained. Each molecule was standardized by salt stripping and neutralization using RDKit, and the parent structure was represented by a canonical isomeric SMILES string and InChIKey. The resulting FDA-approved parent structures were cross-annotated against ChEMBL `max_phase = 4` small molecules as an orthogonal corroboration layer, but ChEMBL was not used as the primary approval source. The first-pass screening subset excluded isotopically labeled compounds and molecules containing nonclassical elements, then applied a soft physicochemical envelope (`MW 120-700`, `cLogP -1 to 6`, `HBD <= 7`, `HBA <= 12`, `RB <= 12`, `TPSA <= 160`) together with PAINS exclusion. Brenk alerts were retained as prioritization flags rather than hard exclusions because the library was already restricted to clinically approved molecules. A high-confidence subset was defined by the additional requirement of ChEMBL support. Target-specific subsets were then generated for the `L` polymerase pocket and for the more restrictive `F` prefusion pocket.
