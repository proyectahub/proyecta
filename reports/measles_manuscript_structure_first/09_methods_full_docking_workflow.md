## Integrated docking and refinement workflow

### Compound library assembly

A docking-oriented repurposing library was assembled from the DrugCentral `FDA_Approved.csv` registry and the DrugCentral 2023 structural archive. Only compounds with an explicit recoverable chemical structure were retained. Each molecule was standardized by salt stripping and neutralization using RDKit, represented by a canonical isomeric SMILES string and InChIKey, and cross-annotated against ChEMBL `max_phase = 4` small molecules as an orthogonal corroboration layer. The first-pass screening subset excluded isotopically labeled compounds and molecules containing nonclassical elements, then applied a soft physicochemical envelope (`MW 120-700`, `cLogP -1 to 6`, `HBD <= 7`, `HBA <= 12`, `RB <= 12`, `TPSA <= 160`) together with PAINS exclusion. Brenk alerts were retained as prioritization flags rather than hard exclusions because the chemical universe had already been restricted to clinically approved molecules.

### First-pass structure-anchored docking boxes

The first-pass `L` polymerase docking campaign was anchored to the ERDRP-0519-bound `9OCF` holo structure. The initial search box was derived from the full bound ligand pose and centered at `(170.516, 167.880, 187.170)` with dimensions `21.099 x 13.600 x 24.173 A`. The first-pass `F` prefusion campaign was anchored to the AS-48-bound `5YZC` structure. The corresponding initial search box was derived from the full AS-48 pose and centered at `(-21.322, -19.395, -6.043)` with dimensions `15.015 x 13.805 x 17.617 A`. These broader ligand-defined boxes were retained for discovery-oriented screening because they preserve the experimentally observed target-engaged pocket identity while allowing limited geometric flexibility for approved molecules that differ from the reference ligands.

### Conservation-guided refinement of the L polymerase pocket

To align the polymerase workflow with the sequence analysis, the 31 residues located within `6 A` of ERDRP-0519 in `9OCF` were intersected with the pocket-level conservation profile derived from the curated regional-global `L` alignment. Residues were retained in the conserved structural core only when they satisfied both a minimum ligand-contact distance of `<=4 A` and a consensus frequency of `1.0` across the non-gap alignment positions. This yielded a 15-residue conserved core comprising `LEU664`, `TYR667`, `TRP671`, `GLU740`, `GLY741`, `GLN744`, `TRP747`, `THR751`, `LEU755`, `GLY772`, `ASP773`, `GLN775`, `LEU811`, `HIS816`, and `LEU818`. ERDRP-0519 atoms located within `4 A` of this conserved core were then used to derive a stricter refinement box centered at `(172.236, 168.128, 188.918)` with final dimensions `17.033 x 12.000 x 17.607 A`, corresponding to a `3 A` margin per side and a minimum edge length of `12 A`.

### Conservation-guided refinement of the prefusion F pocket

An analogous procedure was applied to the prefusion `F` pocket. The 13 residues within `6 A` of AS-48 in `5YZC` were intersected with the curated regional-global `F` alignment. Residues were retained in the conserved core only when they satisfied both a minimum contact distance of `<=4 A` and a consensus frequency of `1.0`. This yielded a 7-residue invariant core comprising `TRP311`, `THR369`, `LEU370`, `PRO451`, `ILE452`, `VAL459`, and `ASN462`. All AS-48 atoms remained within `4 A` of this invariant core. A stricter refinement box was therefore defined at `(-21.322, -19.395, -6.043)` with dimensions `13.015 x 12.000 x 15.617 A`, again using a `3 A` margin per side and a minimum edge length of `12 A`.

### First-pass screening, incremental monitoring, and post-screen refinement

The current `L` campaign was executed as a first-pass virtual screen over the broader `9OCF` ligand-defined search box using a tier-1 FDA-approved subset together with structural controls. Runs were monitored through a remote progress interface that reports ligand generation, docking poses, logs, and incrementally written result rows. Because the first-pass run now writes partial CSV output, the ranked candidate list can be inspected during execution; however, automated post-screen refinement is triggered only after completion of the full first-pass run by default. At completion, the remote bundle is synchronized locally, the top-ranked candidate set is extracted from the first-pass results, and a second redocking campaign is launched in the conserved-core refinement box. In the current configuration, the automated post-screen script is set to carry forward the top `50` candidate ligands together with all structural controls into the conserved-core `L` refinement campaign.

### Rationale for the two-level docking design

This workflow should be described as `structure-anchored, conservation-validated docking`. The primary search region is defined by experimentally observed ligand engagement in the holo structure, whereas the secondary refinement stage is defined by the invariant interaction nucleus identified by the sequence alignment. Under this design, the first-pass screen is not a blind search across the entire protein surface, but neither is it restricted prematurely to a rigid microcavity. Instead, discovery is allowed within an experimentally supported pocket neighborhood, and subsequent redocking or rescoring is focused on the most conserved structural core of that pocket.
